---

**김진웅 즉 Host로 어떻게 접속하냐**
지금 구현 기준으로는 **쿠키 로그인 방식이 아니라, create 응답으로 받은 `userId`를 FE가 들고 가는 구조**예요.

그래서 host가 되는 방법은 2가지예요.

**방법 1. 가장 정상적인 방법**
- FE에서 `방 만들기`를 누름
- `POST /rooms` 성공
- 응답으로 받은 `userId`가 host userId가 됨
- 그 상태로 Lobby에 들어가면 host로 동작

즉, **방을 직접 만든 사람이 host**입니다.

**방법 2. DB에 미리 넣어둔 김진웅(host) 방으로 “host처럼” 접속하고 싶을 때**
지금 구조상 FE는 DB만 보고 host가 되는 게 아니라,
**현재 FE가 들고 있는 `currentUserId`가 rooms.host_user_id와 같아야 host처럼 동작**합니다.

즉 seeded DB에 아래처럼 들어있다면:

- host nickname: `김진웅`
- host user id: `user-host-kim`
- room code: `AB12CD`

FE도 현재 유저를 `user-host-kim`으로 들고 있어야 host로 인식됩니다.

하지만 지금 FE는:
- create/join API 성공 응답으로 받은 `userId`를 사용함

그래서 **이미 DB에 넣어둔 김진웅(host) 방에 그냥 join 하면 host가 아니라 새 participant가 됩니다.**

즉 아주 솔직히 말하면:

- **host로 가장 쉽게 테스트하려면 FE에서 새로 `방 만들기`를 하는 게 맞고**
- **DB seed로 넣어둔 김진웅을 host로 FE에서 바로 접속하는 기능은 아직 따로 없습니다.**

**짧게 요약**
- DB 확인은 `SELECT * FROM users;`, `SELECT * FROM rooms;`, `SELECT * FROM room_participants;`
- 김진웅 host 확인은 `rooms.host_user_id`와 `users.nickname='김진웅'` JOIN으로 보면 됨
- FE에서 host로 들어가려면 **현재 구조상 그 host의 `userId`를 FE 세션이 들고 있어야 함**
- 그래서 지금은 **직접 방 만들기 한 사람이 host로 테스트하는 게 가장 쉬움**
